## Sovereign Process Architecture (SPA) — the methodology

SPA is a domain-agnostic specification grammar built on four architectural invariants:

- **Provenance-First Verification** — no inference enters the reasoning chain without a traceable source
- **Gate-Ordered Reasoning** — later gates cannot bypass earlier requirements; sequence is enforced architecturally
- **Longitudinal Window-Based Truth** — truth is a governed trajectory, not a snapshot
- **Deterministic Validation with Prohibited Content Enforcement** — specific outputs are structurally blocked at the architecture level, not the policy level

Every SPA-derived system produces an **immutable Flight Recorder** — a SHA-256 hash-chain verified audit trail. The reasoning chain is reproducible, traceable, and defensible.
